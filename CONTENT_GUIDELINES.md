# 학습 콘텐츠 작성 가이드라인

> 이 문서는 AI 신약개발 학습 프로그램의 모든 일일 콘텐츠가 따라야 할 규칙, 원칙, 구조를 정의합니다.
> Claude 루틴이 콘텐츠를 자동 생성할 때 이 문서를 참조합니다.

---

## 1. 콘텐츠 철학

### 1.1 핵심 전제
- 학습자는 **풀스택 창업자**를 목표로 한다. 기술과 비즈니스를 동시에 이해해야 한다.
- 코딩은 **바이브코딩**(AI 코딩 도구)으로 해결한다. 코드를 한 줄씩 외우는 것이 아니라, **무엇을 만들어야 하는지, 왜 그렇게 작동하는지**를 이해하는 것이 목적이다.
- 기초를 **탄탄하게** 쌓는다. 피상적 트렌드 나열이 아니라, 원리를 깊이 이해해서 새로운 기술이 나와도 스스로 판단할 수 있는 능력을 기른다.

### 1.2 콘텐츠가 답해야 할 질문
매 Day의 콘텐츠는 반드시 다음 질문들에 답해야 한다:

1. **이것이 무엇인가?** — 정의와 핵심 개념
2. **왜 존재하는가?** — 해결하는 문제, 이전 방법의 한계
3. **어떻게 작동하는가?** — 내부 메커니즘 (코드가 아닌 원리 수준)
4. **어디에 쓰이는가?** — 신약개발에서의 구체적 활용
5. **창업에 어떻게 연결되는가?** — 제품/서비스 가능성, 시장 가치

### 1.3 금지 사항
- 코드 중심 튜토리얼 금지. 코드는 "이런 구조로 만들어라"는 아키텍처 의사코드만 허용.
- 근거 없는 낙관("이 기술이 혁명을 일으킬 것") 금지. 항상 근거(논문/데이터)와 함께.
- 나열식 정보 금지. 항상 "그래서 뭐?"(So what?)를 답해야 한다.
- 오래된 정보 금지. 2023년 이전 정보는 역사적 맥락으로만 사용.

## 2. 대상 독자 프로필

| 항목 | 내용 |
|------|------|
| 전공 | **약학** (약물의 작용 기전, 약동학, 제형, 임상시험 설계에 대한 전문 지식 보유) |
| 도메인 강점 | 약리학, 약물동태학(PK/PD), ADME, 독성학, 임상시험 프로세스, 규제(FDA/MFDS) |
| 코딩 | Python 중급, 바이브코딩(Claude Code 등)으로 프로덕트 구축 가능 |
| AI/ML | 개념적 이해 필요, 수학적 증명 불필요 |
| 목표 | AI 신약개발 스타트업 창업 (풀스택 — 기술+비즈니스) |
| 학습 시간 | 하루 3-4시간 |
| 학습 방식 | 읽기 → 이해 → 리서치 → 분석 (코딩 실습 아님) |

### 2.1 약학 전공자의 이점을 활용하는 방법
- **ADMET 설명 시**: "약사라면 이미 아는 약동학 개념을 AI가 어떻게 모델링하는지" 관점으로 연결
- **타겟 발굴 시**: 약리학적 기전 이해를 전제하고, AI가 이를 어떻게 가속하는지에 집중
- **임상 설계 시**: 규제 요구사항을 알고 있으므로, AI 예측 결과를 규제 관점에서 해석
- **제형/전달 시**: DDS(Drug Delivery System) 지식과 AI 최적화의 결합 가능성
- **비유 선택 시**: 생화학적 비유보다 **약학적 비유** 우선 (예: "모델의 일반화 = 약물의 적응증 확장")

## 3. 일일 콘텐츠 구조

각 Day는 아래 6개 섹션으로 구성된다. **총 분량: 2,500~4,000자 (한국어 기준)**

### 섹션 구조

```markdown
# Day N: [주제명]

## 개요
(3-5줄. 이 주제가 무엇이고 왜 중요한지 한 문단 요약)

## 핵심 개념
(이 주제의 기초 이론. 정의, 원리, 메커니즘. 학술적 톤.)

## 작동 원리와 아키텍처
(어떻게 구현되는지. 코드가 아닌 시스템 설계 수준.
 바이브코딩 시 AI에게 지시할 때 필요한 구조적 이해.)

## 신약개발 적용
(이 기술이 실제로 신약개발 어디에서 사용되는지.
 구체적 사례, 회사, 논문, 성과.)

## 창업 관점
(이 기술로 어떤 제품/서비스를 만들 수 있는지.
 시장, 경쟁, 차별화, 비즈니스 모델.)

## 오늘의 과제
(실행 가능한 리서치/분석/설계 과제 2-3개)

## 참고 자료
(논문, 회사, 도구 링크)
```

## 4. 섹션별 작성 규칙

### 4.1 개요
- 3-5문장. 첫 문장은 정의, 나머지는 "왜 중요한가".
- 이 섹션만 읽어도 오늘 뭘 배우는지 알 수 있어야 한다.
- 예시: "ADMET 예측은 약물 후보의 흡수·분포·대사·배설·독성을 컴퓨터로 예측하는 기술이다. 임상 실패의 60%가 ADMET 문제에서 발생하며, 이를 초기에 예측하면 수억 달러의 실패 비용을 절감할 수 있다."

### 4.2 핵심 개념
- 학술적 톤. 교과서처럼 정확하고 체계적으로.
- 핵심 용어는 **볼드**로 표시하고 처음 등장 시 정의.
- 수식이 필요하면 LaTeX가 아닌 **말로 설명** + 직관적 비유.
- **약학 전공 지식을 적극 활용**하여 연결. (예: "GNN이 분자 구조에서 활성을 예측하는 것은, 약사가 구조-활성 관계(SAR)를 읽는 것의 자동화이다")
- 약동학/약력학 개념과 AI 모델의 대응 관계를 명시. (예: "ADMET 예측 모델의 출력 = 약사가 약물 상호작용 검토 시 확인하는 파라미터")
- 역사적 맥락 간략 포함 (이전에 어떻게 했는지 → 왜 새 방법이 필요했는지)

### 4.3 작동 원리와 아키텍처
- **입력 → 처리 → 출력** 흐름을 명확히.
- 시스템 구성요소를 나열하고 각 역할을 설명.
- 코드가 아닌 **아키텍처 다이어그램 수준**의 설명.
- "바이브코딩으로 이걸 만들려면 AI에게 이렇게 지시한다" 수준의 의사코드는 허용:

```
[허용되는 의사코드 예시]
시스템 구성:
1. 분자 입력 모듈: SMILES → 분자 그래프 변환
2. GNN 인코더: 3-layer message passing → 분자 임베딩(256차원)
3. 물성 예측 헤드: 임베딩 → FC layer → pIC50 값 출력
4. 불확실성 모듈: MC Dropout으로 예측 신뢰구간 제공
```

- 핵심 파라미터와 설계 결정 사항을 표로 정리:

```
| 설계 결정 | 선택지 | 추천 | 이유 |
|-----------|--------|------|------|
| 분자 표현 | SMILES / Graph / 3D | Graph | 구조 정보 보존 |
| GNN 종류 | GCN / GAT / MPNN | MPNN | 엣지 특성 활용 가능 |
```

### 4.4 신약개발 적용
- **구체적 사례** 필수. "~에 활용된다" (X) → "Insilico Medicine이 2023년 이 기술로 IPF 치료제 후보를 18개월 만에 발굴했다" (O)
- 가능하면 **수치** 포함 (비용, 시간, 성공률)
- 기존 방법 대비 개선점을 비교표로

### 4.5 창업 관점
- **제품 아이디어**: 이 기술로 어떤 SaaS/플랫폼/도구를 만들 수 있는가
- **시장 규모**: TAM/SAM/SOM 또는 관련 시장 데이터
- **경쟁 구도**: 이 분야의 기존 플레이어 (스타트업 + 빅파마 + 빅테크)
- **기술적 해자(moat)**: 어떻게 차별화할 수 있는가
- **MVP 구상**: 바이브코딩으로 최소 기능 제품을 빠르게 만든다면?

### 4.6 오늘의 과제
- **2-3개** 실행 과제. 각 과제는 30-60분 분량.
- 과제 유형 (매일 조합):

| 유형 | 설명 | 예시 |
|------|------|------|
| 기초 학습 | 핵심 개념 깊이 파기 | "MPNN 논문 Abstract 읽고 핵심 3줄 요약" |
| 비즈니스 분석 | 시장/경쟁/가치 분석 | "ADMET 예측 스타트업 3곳 비교 분석" |
| 리서치 정리 | 논문/기사 읽고 정리 | "AlphaFold3 임팩트 1페이지 정리" |

- 과제는 **구체적이고 완료 가능**해야 한다. "~에 대해 생각해보기" (X) → "~에 대해 1페이지 문서 작성" (O)

### 4.7 참고 자료
- 필수 논문 1-2개 (제목, 저자, 연도, 한줄 요약)
- 관련 회사/제품 웹사이트 1-2개
- 한국어 자료가 있으면 우선 포함

## 5. 문체와 톤

### 5.1 기본 톤
- **학술적이되 접근 가능한.** 교과서 + 비유의 조합.
- 경어체(~합니다) 사용.
- 객관적 서술 기본. 주관적 의견은 "~로 판단된다", "~가 유력하다"로 표현.

### 5.2 문장 규칙
- 한 문장은 40자 이내 권장. 60자 초과 금지.
- 전문 용어는 첫 등장 시 **한국어(영어)** 병기. 예: **그래프 신경망(Graph Neural Network, GNN)**
- 약어는 첫 등장 시 풀어쓰고 이후 약어만 사용.

### 5.3 구조화 규칙
- 3줄 이상 나열이면 반드시 글머리 기호 또는 표 사용.
- 비교가 필요하면 반드시 표 사용.
- 중요 포인트는 `> 인용 블록`으로 강조.

## 6. 퀴즈 작성 규칙

각 Week당 10문제. 아래 규칙을 따른다:

### 6.1 문제 유형 비율
- 개념 확인 (정의/원리): 4문제
- 적용/판단 (상황에서 올바른 선택): 3문제
- 비즈니스/전략 (창업 관점 판단): 3문제

### 6.2 형식
```yaml
- id: "wNqM"
  question: "질문 (상황 제시 → 판단 요구)"
  type: "multiple_choice"
  options: ["선택지1", "선택지2", "선택지3", "선택지4"]
  answer: "0"  # 0-indexed 정답
  explanation: "왜 이것이 정답인지 + 오답이 왜 틀린지 1-2문장"
```

### 6.3 좋은 퀴즈의 기준
- 단순 암기 X → 이해와 적용을 테스트
- "다음 중 틀린 것은?" 지양 → "가장 적합한 것은?" 선호
- 실제 창업 상황에서의 의사결정을 반영

## 8. 커리큘럼 설계 원칙

### 8.1 주제 선정 기준
1. **창업 필수도**: 이 기술 없이 AI 신약개발 스타트업이 가능한가?
2. **시장 가치**: 이 기술에 기반한 제품의 시장 규모는?
3. **학습 가성비**: 투자 시간 대비 이해도 향상이 큰가?
4. **선행 관계**: 이것을 이해해야 다음 주제를 이해할 수 있는가?

### 8.2 Phase 구성 원칙
- 각 Phase는 하나의 **대주제**를 다룬다.
- Phase 내 Week은 대주제의 **하위 영역**이다.
- Week 내 Day는 하위 영역의 **개별 토픽**이다.
- 마지막 Day는 해당 Week의 **종합 + 창업 적용**이다.

### 8.3 난이도 곡선과 커리큘럼 맵

```
Phase 1: AI 기초 & 전체 그림
  - AI/ML의 핵심 개념 (지도학습, 비지도학습, 딥러닝이 뭔지)
  - 신약개발 파이프라인 전체 조망 (타겟→히트→리드→전임상→임상)
  - AI가 각 단계에서 어떤 역할을 하는지 매핑

Phase 2: 핵심 기술 심화
  - 분자 표현과 화학정보학 (SMILES, 그래프, 지문)
  - 분자 물성 예측 (QSAR, GNN, Transformer)
  - 분자 생성과 최적화 (생성모델, 강화학습)
  - ADMET/독성 예측 (약학 전공 강점 영역)

Phase 3: 최신 기술 & 트렌드
  - 단백질 구조 예측 (AlphaFold, ESM)
  - AI 기반 도킹/MD (DiffDock, 가속 시뮬레이션)
  - LLM/Foundation Model의 신약개발 적용
  - 최신 트렌드: 멀티모달, AI 에이전트, 자동화 실험실

Phase 4: 실무 도구 & 실행
  - 바이브코딩으로 AI 도구 구축 (Claude Code, Cursor 활용법)
  - 데이터 파이프라인 설계 (ChEMBL, PDB, 실험 데이터 통합)
  - MLOps & 배포 (모델 서빙, API, 웹앱)
  - 오픈소스 도구 생태계 (RDKit, PyG, HuggingFace)

Phase 5: 창업 & 비즈니스
  - AI 신약개발 시장 분석 (경쟁사, 투자 동향)
  - 비즈니스 모델 설계 (SaaS, 플랫폼, 파트너십)
  - MVP → PMF → 스케일링 전략
  - 규제/IP 전략, 팀 구성, 피칭
```

### 8.4 AI 실무 도구 활용 (Phase 4에서 다루되 전체적으로 반영)
학습 과정 전반에 걸쳐 다음 도구의 활용법을 자연스럽게 녹인다:

| 도구 | 활용 맥락 |
|------|----------|
| **Claude Code** | 프로토타입 구축, 코드 생성, 논문 분석 요청 |
| **Cursor/Windsurf** | 로컬 프로젝트 바이브코딩 |
| **HuggingFace** | 사전학습 모델 활용, 파인튜닝 |
| **Ollama/Gemma** | 로컬 LLM으로 화학 에이전트 구축 |
| **GitHub Copilot** | 일상적 코딩 보조 |
