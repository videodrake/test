# 매일 학습 콘텐츠 업데이트 루틴

아래 절차에 따라 학습 콘텐츠를 1일치 업데이트합니다.

## 1단계: 현재 위치 파악

`data/progress.json`을 읽어 현재 `current_phase`, `current_week`, `current_day`를 확인합니다.
`curriculum/overview.yaml`에서 해당 Day의 `title`과 `content_path`를 찾습니다.

## 2단계: 이전 Day 콘텐츠 확인

해당 Day가 Week의 첫째 Day(Day 1)가 아니면, 바로 이전 Day의 .md 파일을 읽어 연결점을 파악합니다.
Week의 첫째 Day면 이전 Week 마지막 Day(Day 5)를 확인합니다.

## 3단계: 콘텐츠 작성

`CONTENT_GUIDELINES.md`를 읽고, 해당 Day의 .md 파일을 아래 구조로 **완전히 새로 작성**합니다.

### 콘텐츠 구조 (4,000~6,000자)

```markdown
# Day N: [overview.yaml의 title]

> 이전 학습에서 [X]를 다뤘습니다. 오늘은 이를 바탕으로 [Y]를 학습합니다.

## 개요
(3-5문장. 정의 + 왜 중요한지. 이 섹션만 읽어도 오늘 뭘 배우는지 알 수 있어야 함.)

## 핵심 개념
(학술적 톤. 용어 정의, 원리, 메커니즘. 약학 지식과 대응 관계 명시.
 코드가 아닌 개념 수준. 역사적 맥락 포함.)

## 작동 원리와 아키텍처
(입력→처리→출력 흐름. 시스템 구성요소.
 바이브코딩 시 AI에게 지시할 때 필요한 구조적 이해.
 의사코드/설계표 허용.)

## 신약개발 적용
(구체적 사례 필수 — 회사명, 연도, 수치.
 기존 방법 대비 개선점 비교.)

## 창업 관점
(Phase별 비중 조절:
 Phase 1→간략 1-2문장,
 Phase 2-3→제품 아이디어+경쟁 구도,
 Phase 4-5→시장규모+BM+MVP 구상)

## 오늘의 과제
(2-3개. 구체적이고 30-60분 내 완료 가능.)
- 기초 학습: 논문 Abstract 읽기, 개념 정리 등
- 비즈니스 분석: 경쟁사 조사, 시장 분석 등
- 리서치 정리: 핵심 인사이트 1페이지 문서 작성 등

## 참고 자료
- 논문 1-2개 (제목, 저자, 연도, 한줄 요약)
- 관련 회사/도구 1-2개
```

### 작성 시 핵심 규칙
- **코드 블록 최소화** (3개 이하, 의사코드/아키텍처만)
- **약학 전공 지식 활용** — 단순 비유가 아닌 실제 경쟁력 연결
- **구체적 사례와 수치** 최소 2개 포함
- **학술적이되 접근 가능한 톤** — 경어체, 용어 한영 병기
- 최신 정보가 필요하면 **WebSearch로 보강** (확인 불가 시 "확인 필요" 표기)

## 4단계: 퀴즈 업데이트 (Week의 마지막 Day일 때만)

해당 Day가 Day 5(Week 마지막)이면, 같은 Week의 quiz yaml을 작성/업데이트합니다.
10문제, 형식:

```yaml
title: "Week N: [주제] 퀴즈"
description: "설명"
questions:
  - id: "wNqM"
    question: "질문"
    type: "multiple_choice"
    options: ["A", "B", "C", "D"]
    answer: "0"
    explanation: "해설"
```

문제 비율: 개념 확인 4 + 적용/판단 3 + 비즈니스/전략 3

## 5단계: 빌드와 배포

```bash
python build.py
git add -A
git commit -m "Day [N] 콘텐츠 업데이트: [Day 제목]"
git push
```

## 6단계: 품질 자체 검증

작성 후 체크:
- [ ] 6개 섹션 모두 존재
- [ ] 4,000~6,000자 범위
- [ ] 구체적 사례/수치 2개 이상
- [ ] 코드 블록 3개 이하
- [ ] 이전 Day와 연결문 존재
- [ ] 과제가 구체적이고 실행 가능
