# Day 1: AI란 무엇인가 — 약학 전공자를 위한 AI 입문

> 이 프로그램의 첫 학습입니다. 오늘은 AI의 기본 정의와 작동 원리를 약학적 맥락에서 이해하고, 이 기술이 신약개발의 어떤 문제를 해결하는지 전체 그림을 그립니다.

## 개요

**인공지능(Artificial Intelligence, AI)**은 인간의 학습, 추론, 판단 능력을 컴퓨터로 구현하는 기술입니다. 신약개발에서 AI는 후보 물질 탐색에 평균 4.5년, 26억 달러가 소요되는 과정을 획기적으로 단축할 수 있는 도구로 주목받고 있습니다. 약학 전공자에게 AI는 단순한 도구가 아니라, 자신이 이미 보유한 약리학·PK/PD·독성학 지식을 **대규모로 확장하는 레버리지**입니다. 약사가 처방전 100장을 검토하는 것과 AI가 100만 개 화합물을 스크리닝하는 것은 본질적으로 같은 판단 — "이 물질이 안전하고 효과적인가?" — 의 스케일 차이입니다.

## 핵심 개념

### AI, 머신러닝, 딥러닝의 관계

이 세 용어는 종종 혼용되지만, 포함 관계에 있습니다.

**인공지능(AI)**은 가장 넓은 개념으로, 컴퓨터가 인간처럼 지능적 작업을 수행하는 모든 기술을 포괄합니다. 규칙 기반 시스템(if-then)부터 최신 대규모 언어 모델까지 모두 AI입니다.

**머신러닝(Machine Learning, ML)**은 AI의 하위 분야로, 명시적 규칙 대신 **데이터로부터 패턴을 학습**하는 접근법입니다. 약학에서 비유하면, 약전(규칙 기반)과 임상 데이터 분석(데이터 기반)의 차이와 같습니다. 전통적 프로그래밍은 "logP > 5이면 흡수 불량"이라는 규칙을 사람이 작성하지만, ML은 수만 개의 흡수 데이터에서 이 규칙을 **스스로 발견**합니다.

**딥러닝(Deep Learning, DL)**은 ML의 하위 분야로, 여러 층의 **인공 신경망(Neural Network)**을 사용합니다. 분자 구조처럼 복잡한 비선형 관계를 포착하는 데 강력합니다. 2012년 ImageNet 대회에서 딥러닝이 기존 방법을 압도한 이후, 의약학을 포함한 거의 모든 분야에 확산되었습니다.

```
AI (인공지능)
 └── ML (머신러닝) — 데이터에서 패턴 학습
      └── DL (딥러닝) — 다층 신경망
           └── Transformer, GNN, ... — 최신 아키텍처
```

### 약학에서 이미 쓰고 있는 "AI적 사고"

약학 전공자는 이미 AI의 핵심 원리를 직관적으로 이해하고 있습니다:

| 약학 개념 | AI 대응 개념 | 연결 |
|-----------|-------------|------|
| **구조-활성 관계(SAR)** | 지도학습(Supervised Learning) | 분자 구조(입력)로 활성(출력)을 예측 |
| **Lipinski's Rule of Five** | 분류 규칙(Decision Boundary) | 약물 유사성의 기준선 |
| **용량-반응 곡선** | 회귀(Regression) | 연속적 값을 예측하는 모델 |
| **약물 상호작용 검토** | 다중 입력 모델 | 여러 변수를 동시에 고려 |
| **처방 검토 시 패턴 인식** | 패턴 매칭(Pattern Recognition) | 경험에서 규칙을 추출 |

> **핵심 인사이트**: AI는 약사가 이미 하는 일 — 구조를 보고 활성을 예측하고, 부작용을 판단하고, 상호작용을 검토하는 일 — 을 수백만 배 스케일로 수행합니다. 차이는 속도와 규모이지 본질이 아닙니다.

### AI가 해결하는 신약개발의 근본 문제

전통적 신약개발의 핵심 병목:

| 문제 | 현재 상황 | AI의 기여 |
|------|----------|----------|
| **탐색 공간** | 합성 가능한 분자 ~10^60개, 인간이 탐색 불가 | 가상 스크리닝으로 수십억 개를 몇 시간에 평가 |
| **실패율** | 임상 진입 약물의 ~90%가 최종 실패 | ADMET/독성 조기 예측으로 실패 후보 사전 제거 |
| **비용** | 신약 1개에 평균 26억 달러 (Tufts CSDD, 2020) | 실험 횟수 감소, 사이클 단축 |
| **시간** | 타겟에서 시판까지 10-15년 | Insilico Medicine: 타겟→임상 Phase II를 30개월에 달성 |
| **데이터 해석** | 방대한 오믹스/스크리닝 데이터를 수동 분석 | 패턴 자동 발견, 바이오마커 탐색 |

## 작동 원리와 아키텍처

### ML 모델의 기본 작동 흐름

어떤 ML 모델이든 기본 흐름은 동일합니다:

```
[데이터 수집] → [전처리] → [학습] → [예측] → [평가]

신약개발 예시:
ChEMBL에서        분자를 숫자로    패턴 발견    새 분자의     실험으로
활성 데이터 수집 → 변환(기술자) → (모델 학습) → 활성 예측 → 검증
```

### 바이브코딩 관점: AI 시스템의 구성요소

바이브코딩으로 AI 도구를 만들 때, AI에게 지시해야 하는 핵심 구성요소:

```
시스템 구성 (예: 용해도 예측 서비스):
1. 데이터 모듈: ChEMBL API에서 용해도 데이터 수집 + 전처리
2. 분자 표현: SMILES → 분자 기술자 변환 (MW, logP, TPSA 등)
3. 모델: Random Forest 또는 GNN으로 학습
4. API: 사용자가 SMILES를 입력하면 예측 용해도 반환
5. UI: 웹 대시보드로 결과 시각화
```

| 구성요소 | 당신이 결정할 것 | AI 도구가 구현할 것 |
|---------|----------------|-------------------|
| 데이터 | 어떤 데이터가 필요한지, 품질 기준 | 수집/정제 코드 |
| 표현 | 어떤 분자 표현이 적합한지 | 변환 파이프라인 |
| 모델 | 어떤 알고리즘이 문제에 맞는지 | 학습/최적화 코드 |
| 평가 | 무엇이 "좋은 성능"인지의 기준 | 평가 메트릭 계산 |

> 약학 전공자의 역할은 **"무엇을"과 "왜"를 결정**하는 것입니다. "어떻게"는 Claude Code가 합니다.

## 신약개발 적용

### 실제 AI 신약개발 성과

**Insilico Medicine** (2024): AI 플랫폼 Pharma.AI로 특발성 폐섬유증(IPF) 치료제 후보 INS018_055를 발굴하여 Phase II 임상에 진입했습니다. 타겟 발굴부터 전임상까지 30개월이 소요되었으며, 이는 전통적 방법 대비 약 4배 빠른 속도입니다. 전체 비용은 260만 달러로 보고되었습니다.

**Recursion Pharmaceuticals** (2024): 자체 구축한 대규모 세포 이미지 데이터셋(2,400만+ 실험)과 AI를 결합하여, 희귀질환과 종양학 분야에서 다수의 파이프라인을 운영하고 있습니다. 2024년 기준 시가총액 약 30억 달러.

| 방법 | 타겟→후보 물질 | 비용 | 성공률 |
|------|-------------|------|-------|
| 전통적 | 4-5년 | 수억 달러 | ~5% |
| AI 가속 | 1-2년 | 수백만 달러 | 아직 검증 중 |

## 창업 관점

AI 신약개발은 현재 급성장 중인 시장입니다. 이 프로그램을 통해 기술과 비즈니스를 동시에 이해함으로써, 약학 전공자로서 독보적인 포지션을 구축할 수 있습니다. 구체적인 시장 분석과 창업 전략은 Phase 5에서 본격적으로 다룹니다.
