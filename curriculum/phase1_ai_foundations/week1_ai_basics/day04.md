# Day 4: AI 모델의 학습 원리 — 데이터, 손실함수, 최적화

> 이전 학습에서 단일 뉴런부터 Transformer까지 딥러닝 아키텍처의 진화와 학습 루프 4단계(forward → loss → backward → update)를 다뤘습니다. 오늘은 그 학습 루프를 구성하는 세 기둥, **데이터(data)**·**손실함수(loss function)**·**최적화(optimization)** 의 내부를 들여다보고, 약학적 데이터 제약 아래에서 모델이 일반화에 성공하는(또는 실패하는) 이유를 학습합니다.

## 개요

AI 모델의 성능을 결정하는 것은 아키텍처의 화려함이 아니라, **무엇을 학습 신호로 정의하고, 어떻게 그 신호를 따라가도록 만드느냐** 입니다. 학습 신호의 원천이 데이터, 신호의 형태가 손실함수, 신호를 따라가는 절차가 최적화입니다. 신약개발 데이터는 일반 ML 데이터와 본질적으로 다릅니다. 라벨은 수백~수천 건으로 적고, 분포는 활성 화합물에 편향되며, 스캐폴드(scaffold)별로 강한 군집을 이룹니다. 이 제약을 모르면 RMSE 0.4의 우수한 모델이 신규 scaffold에서는 무작위와 다를 바 없는 사태가 발생합니다. 오늘의 목표는 데이터 분할·손실 설계·최적화 알고리즘 각각의 원리와 함정을 이해해, 모델 결과가 신뢰할 만한지를 약학적 관점에서 판단하는 능력을 기르는 것입니다.

## 핵심 개념

### 데이터 — 학습의 출발점이자 천장

머신러닝에서 모델이 도달할 수 있는 성능 상한은 데이터의 질과 분할 방식이 결정합니다. 데이터셋은 일반적으로 **훈련(train)**·**검증(validation)**·**시험(test)** 셋으로 나누며, 각 역할은 다음과 같습니다.

| 분할 | 역할 | 약학적 비유 |
|------|------|-----------|
| Train | 파라미터 학습 | in vitro 어세이 데이터셋 |
| Validation | 하이퍼파라미터 선택, 조기 종료 | 내부 검증 코호트 |
| Test | 최종 일반화 평가 (한 번만 사용) | 외부 임상 검증 |

분할 방식에 따라 보고되는 성능이 극적으로 달라집니다. **무작위 분할(random split)** 은 같은 scaffold의 유사 분자가 train과 test에 섞여 성능을 과대평가합니다. **스캐폴드 분할(scaffold split)** 은 Bemis-Murcko scaffold 단위로 train/test를 나눠 "처음 보는 분자 골격으로의 외삽" 능력을 측정합니다. MoleculeNet 벤치마크에서 동일 모델이 random split에서는 ROC-AUC 0.90을, scaffold split에서는 0.72를 기록하는 사례가 흔합니다. 약물 탐색은 본질적으로 외삽 문제이므로, 스캐폴드 분할이 임상 현실을 더 잘 반영합니다.

데이터의 또 다른 축은 **적용 영역(applicability domain, AD)** 입니다. QSAR에서 오래 다뤄온 개념으로, 모델은 훈련 데이터의 화학공간 내부에서만 신뢰할 수 있다는 원칙입니다. AD를 무시한 예측은 외삽이며, 신뢰구간 없이 점추정만 보고되면 의사결정이 오도될 수 있습니다.

### 손실함수 — 모델에게 무엇을 향상시킬지 알려주는 신호

**손실함수** 는 예측값 ŷ과 실측값 y의 괴리를 하나의 스칼라로 요약한 함수입니다. 학습은 이 손실을 최소화하는 방향으로 파라미터를 갱신합니다. 과제 유형별 표준 선택은 다음과 같습니다.

| 과제 | 손실함수 | 수식 개요 | 신약개발 예 |
|------|---------|---------|-----------|
| 회귀 | MSE / MAE | (y−ŷ)² 평균 / |y−ŷ| 평균 | pIC50, logP 예측 |
| 이진 분류 | Binary Cross-Entropy | −[y log ŷ + (1−y) log(1−ŷ)] | hERG 차단 여부 |
| 다중 분류 | Categorical CE | −Σ yᵢ log ŷᵢ | 독성 등급, ATC 분류 |
| 표현 학습 | Contrastive (InfoNCE) | 양·음성 쌍 거리 대비 | SSL 분자 임베딩 |
| 순위 학습 | Pairwise / Listwise | 순서 보존 페널티 | 가상 스크리닝 랭킹 |

손실 선택은 단순 관행이 아니라 **약학적 의사결정** 입니다. 가령 hERG 차단 분류에서 위양성과 위음성의 비용은 비대칭입니다. 위음성(차단을 놓침)은 후기 임상에서 QT 연장으로 개발 중단을 부를 수 있으므로, 양성 클래스에 가중치를 둔 **가중 BCE** 또는 **Focal Loss** 가 단순 BCE보다 합리적입니다. 마찬가지로 IC50 회귀에서 활성 영역(저농도) 정확도가 비활성 영역보다 임상적으로 중요할 수 있으므로, 활성 구간을 강조하는 가중 MSE 또는 로그 변환 후 MSE가 자주 채택됩니다.

### 최적화 — 손실 표면을 따라 내려가는 절차

손실함수는 파라미터 공간 위에 정의된 비볼록(non-convex) 표면이며, **경사하강법(gradient descent)** 은 이 표면의 기울기를 따라 한 걸음씩 내려갑니다. 한 번에 전체 데이터로 기울기를 계산하면 비싸므로, 작은 배치로 근사하는 **확률적 경사하강법(SGD, Stochastic GD)** 과 그 변형이 표준입니다.

| 옵티마이저 | 핵심 아이디어 | 권장 상황 |
|----------|-----------|---------|
| SGD + Momentum | 관성으로 진동 억제 | 대규모 비전·잘 정제된 데이터 |
| Adam | 1·2차 모멘트 적응 학습률 | 신약개발 GNN/Transformer 기본값 |
| AdamW | weight decay 분리 | Transformer 사전학습 표준 |
| Lookahead, LAMB | 대규모 배치 안정화 | 수억 분자 사전학습 |

**학습률(learning rate)** 은 최적화 성능을 좌우하는 가장 민감한 하이퍼파라미터입니다. 너무 크면 발산, 너무 작으면 정체합니다. 실무에서는 warm-up 후 cosine decay 같은 **학습률 스케줄러** 를 거의 항상 사용합니다.

### 일반화의 적 — 과적합과 정규화

**과적합(overfitting)** 은 훈련 손실은 낮지만 시험 손실이 높은 현상으로, 모델이 데이터의 신호 대신 잡음을 외운 상태입니다. 신약개발에서 데이터셋이 작을수록 위험이 커집니다. 표준 대응책은 다음과 같습니다.

- **L2/L1 정규화**: 가중치 크기에 페널티
- **Dropout**: 학습 중 일부 뉴런을 무작위로 비활성화 — 약리학의 redundancy(여러 경로가 동일 표현형 매개)와 닮은 효과
- **조기 종료(early stopping)**: 검증 손실이 개선을 멈추면 학습 중단
- **데이터 증강**: SMILES 임의화, 분자 회전, 노이즈 주입
- **앙상블·MC Dropout**: 예측 불확실성을 함께 보고해 외삽 영역을 식별

## 작동 원리와 아키텍처

### 학습 파이프라인의 표준 구성

```
신약개발 모델 학습 파이프라인:
1. 데이터 수집     : ChEMBL/PubChem (SMILES, label) 추출
2. 정제·정규화     : tautomer 표준화, 중복 제거, 라벨 단위 통일(IC50 → pIC50)
3. 분할            : scaffold split (train 80 / val 10 / test 10)
4. 손실 설계       : 회귀=MSE, 분류=가중 BCE, 다과제=합 또는 동적 가중
5. 옵티마이저       : AdamW (lr=1e-4) + cosine schedule + warm-up 1k step
6. 정규화          : Dropout 0.1~0.3, weight decay 1e-2, early stopping patience=10
7. 평가            : val 기반 best checkpoint, test는 최종 1회만 측정
8. 불확실성        : MC Dropout 또는 Deep Ensemble 5개로 신뢰구간 산출
```

### 설계 결정과 약학적 근거

| 결정 | 흔한 오류 | 약학 전공자 권장 |
|------|---------|----------------|
| 분할 방식 | random split만 사용 | scaffold split 병행, 시간 기반 분할 추가 |
| 손실 가중 | 클래스 불균형 무시 | 양성/위양성 비용 분석 후 가중 또는 Focal |
| 학습률 | 임의 1e-3 | 작은 데이터(<1만)에서는 1e-4, warm-up 사용 |
| 조기 종료 | 단일 시드 결과 보고 | 5 시드 평균과 분산 함께 보고 |
| 예측 출력 | 점추정만 보고 | AD + 불확실성 동반 |

이 결정들은 모두 "임상에서 어떤 의사결정에 쓰일 출력인가?"라는 질문에 종속됩니다. 약학 전공자는 모델러보다 임상 의사결정 맥락을 잘 알기 때문에, 이 단계에서 차별적 설계 권한을 가질 수 있습니다.

## 신약개발 적용

학습 원리에 대한 이해 차이가 실제 결과를 어떻게 가르는지 세 사례로 확인합니다.

- **MoleculeNet 벤치마크 (Wu et al., 2018, *Chemical Science*)**: 17개 데이터셋과 다양한 모델을 비교한 표준 벤치마크. 동일 모델이 random과 scaffold split에서 평균 ROC-AUC 차이가 0.10~0.20 발생함을 정량화했습니다. 분할 방식이 모델 비교 자체를 뒤집을 수 있음을 보여준 분기점입니다.
- **Therapeutics Data Commons (TDC, 2021–현재, Stanford)**: 70개 이상의 표준 과제와 분할을 제공해 재현성 위기를 줄인 인프라. ADMET 예측에서 시간 기반 분할(temporal split)을 권장하며, 이는 약학적으로 후기 발표 화합물에 대한 외삽 성능을 측정합니다.
- **Insilico Medicine의 다중 목표 최적화**: pIC50·합성 가능성·ADMET 점수를 결합한 가중 보상으로 강화학습 기반 분자 생성을 수행. 손실(보상) 설계의 약학적 균형이 INS018_055 후보 발굴 30개월 달성의 핵심 요소로 자리잡았습니다.

기존 통계적 QSAR 대비 현대적 학습 파이프라인의 차이는 다음과 같습니다.

| 항목 | 1990s QSAR | 2020s 딥러닝 파이프라인 |
|------|-----------|--------------------|
| 분할 | 무작위 단일 분할 | scaffold/temporal + 5시드 |
| 손실 | MSE 단일 | 가중 BCE/Focal/다과제 |
| 정규화 | 변수 선택 | Dropout, weight decay, augmentation |
| 출력 | 점추정 | 불확실성 + AD 보고 |

## 창업 관점

Phase 1 단계에서는 간략히 짚습니다. 학습 원리에 대한 깊은 이해는 직접 제품이 되지는 않지만, **모델 신뢰성·데이터 전략·평가 프로토콜** 자체가 B2B 신약개발 SaaS의 차별 포인트가 됩니다. 시장 데이터와 BM 설계는 Phase 5에서 다룹니다.

## 오늘의 과제

1. **분할 방식 영향 정량 비교 (40분)**: TDC 또는 MoleculeNet 문서에서 동일 데이터셋(예: BBBP 또는 hERG)을 random split과 scaffold split으로 평가한 결과를 1개 이상 찾고, ROC-AUC 또는 RMSE 차이를 표로 정리합니다. 이 차이가 약학적으로 어떤 의미를 갖는지 3-5문장으로 해설합니다.
2. **손실함수 설계 시나리오 (40분)**: hERG 차단 분류 모델을 가정하고, ① 양성/음성 비용 비대칭, ② 클래스 불균형(양성 약 10%), ③ 임상 단계별 의사결정 활용을 고려해 적합한 손실(가중 BCE vs Focal vs 비용민감 학습)과 그 이유를 1페이지 메모로 작성합니다.
3. **참고 논문 핵심 정리 (30분)**: Wu *et al.* (2018) MoleculeNet 논문 Abstract와 분할 관련 섹션을 읽고, ① 왜 scaffold split이 도입되었는지, ② 한계는 무엇인지(예: scaffold 정의의 임의성), ③ TDC가 제시한 후속 보완책을 각 2-3줄로 요약합니다.
