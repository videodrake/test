# Day 4: AI 모델의 학습 원리 — 데이터, 손실함수, 최적화

> 이전 학습에서 MLP·CNN·GNN·Transformer·Diffusion 등 주요 딥러닝 아키텍처의 진화를 살폈습니다. 오늘은 어떤 아키텍처를 쓰든 공통으로 작동하는 **학습(training) 메커니즘** — 데이터를 어떻게 준비하고, 무엇을 최소화하며, 어떻게 파라미터를 갱신하는지를 학습합니다.

## 개요

AI 모델의 **학습** 은 "예측이 정답과 얼마나 다른지를 정량화한 뒤, 그 차이를 줄이는 방향으로 파라미터를 점진적으로 조정"하는 과정입니다. 이 과정은 **데이터(Data)**, **손실함수(Loss Function)**, **최적화(Optimization)** 라는 세 축으로 구성되며, 셋 중 어느 하나가 부실해도 모델은 신약개발 현장에서 신뢰할 수 없습니다. 약학 전공자 관점에서 이 세 축은 임상시험의 **모집단 정의·평가지표·용량 적정** 에 정확히 대응합니다. 즉, 누구의 데이터로 학습할지(모집단), 무엇을 좋다고 평가할지(엔드포인트), 어떻게 점진적으로 개선할지(용량 조절)를 결정하는 의사결정의 문제입니다. 오늘은 각 축의 원리와 신약개발 맥락의 함정을 정리하고, 바이브코딩으로 학습 파이프라인을 지시할 때 무엇을 명시해야 하는지 익힙니다.

## 핵심 개념

### 데이터 — 학습은 데이터 분할에서 시작된다

원본 데이터셋은 **훈련(train) / 검증(validation) / 테스트(test)** 셋으로 분할됩니다. 훈련 데이터로 가중치를 학습하고, 검증 데이터로 하이퍼파라미터(학습률·층 수 등)를 조정하며, 테스트 데이터는 마지막 단 한 번 일반화 성능을 보고할 때만 사용합니다. 일반적 비율은 8:1:1 또는 7:1:2입니다.

분할 방법이 모델 평가의 정직성을 결정합니다.

| 분할 방식 | 원리 | 신약개발 맥락 |
|---------|------|-------------|
| Random split | 무작위 행 단위 | 동일 scaffold가 train/test에 섞여 성능 과대평가 |
| **Scaffold split** | Bemis-Murcko 골격 단위로 분리 | 신규 scaffold 일반화 평가 (실전과 유사) |
| Time split | 측정 연도 기준 | 미래 데이터 예측 시뮬레이션 |
| Cluster split | 분자 유사도 군집 분리 | 가장 보수적, 실패율 사전 노출 |

Day 3에서 강조한 scaffold split이 여기서 다시 등장합니다. 약학 전공자는 모델 성능 보고서를 받았을 때 **분할 방식부터 묻는 사람** 이어야 합니다. random split의 R² = 0.85가 scaffold split에서 0.45로 떨어지는 일은 흔합니다.

데이터 품질에서는 **데이터 누수(data leakage)** 도 치명적입니다. 같은 화합물의 거울상 이성질체나 염(salt) 형태가 train/test에 분리되어 들어가면 모델은 학습 시 본 분자를 다시 본 셈이 되어 성능이 부풀려집니다. 표준화·중복 제거(dedup)·이성질체 처리는 약학 지식이 직접 기여하는 영역입니다.

### 손실함수 — 무엇을 최소화하는가

**손실함수** L(ŷ, y)는 모델 예측 ŷ과 정답 y의 차이를 하나의 숫자로 정량화합니다. 이 숫자가 작아질수록 좋다고 정의했기 때문에, 손실함수의 형태가 곧 모델이 추구하는 가치입니다.

| 과제 | 표준 손실 | 수식 직관 | 신약개발 사용처 |
|------|---------|---------|-------------|
| 회귀 | **MSE** (평균제곱오차) | 큰 오차를 제곱으로 더 강하게 벌함 | pIC50, logS 예측 |
| 회귀 (이상치 강건) | **MAE** (평균절대오차) | 모든 오차를 동일 가중 | 노이즈가 많은 측정값 |
| 이진 분류 | **이진 교차엔트로피(BCE)** | 정답 클래스 확률의 −log | hERG 차단 여부 |
| 다중 분류 | **교차엔트로피(CE)** | softmax 출력의 −log | 독성 등급 분류 |
| 불균형 분류 | **Focal Loss** | 쉬운 샘플 가중 낮춤 | 희귀 부작용 예측 |
| 생성 | **Negative Log-Likelihood** | 토큰 확률의 −log | SMILES 생성 모델 |

약학 전공자에게는 손실함수의 선택이 곧 **임상 엔드포인트 정의** 와 같다는 인식이 필요합니다. RMSE를 최소화하는 모델은 "전체 평균 정확도가 높은" 분자를 만들지만, 활성 영역(pIC50 ≥ 8)에서의 정확도가 결정적인 경우에는 가중 손실이나 분류 임계값 기반 손실이 더 적절합니다. 임상에서 1차 평가변수를 잘못 잡으면 약효가 있어도 시험이 실패하듯, 손실함수를 잘못 잡으면 모델이 의미 없는 방향으로 최적화됩니다.

### 최적화 — 어떻게 가중치를 갱신하는가

손실함수는 가중치 공간에서 일종의 지형(landscape)을 이루고, 학습은 이 지형의 골짜기를 찾아 내려가는 과정입니다. 표준 도구는 **경사하강법(Gradient Descent)** 이며, 변형이 다음과 같이 발전했습니다.

| 옵티마이저 | 핵심 아이디어 | 특징 | 권장 사용 |
|----------|-----------|-----|---------|
| SGD | 한 배치당 1회 갱신 | 단순, 기본 학습률 민감 | 이론적 비교용 |
| SGD + Momentum | 이전 갱신 방향 누적 | 진동 감소 | CNN 일부 |
| Adam | 모멘텀 + 적응적 학습률 | 거의 모든 과제의 무난한 기본 | MLP·GNN 기본값 |
| **AdamW** | Adam + 가중치 감쇠 분리 | 일반화가 더 우수 | Transformer·BERT·MolFormer |

핵심 하이퍼파라미터는 다음과 같습니다.

- **학습률(learning rate, η)**: 한 번에 얼마나 크게 가중치를 움직일지. 너무 크면 발산, 너무 작으면 멈춤. 보통 1e-4 ~ 1e-3에서 시작.
- **배치 크기(batch size)**: 한 번 갱신에 사용할 샘플 수. 32~256이 일반적, GPU 메모리와 일반화 성능의 절충.
- **에폭(epoch)**: 전체 훈련 데이터를 한 바퀴 도는 단위. 5~100 사이를 검증 손실로 결정.
- **학습률 스케줄(scheduler)**: 학습이 진행됨에 따라 η를 조정. Warmup + Cosine Decay가 Transformer의 사실상 표준.

> 약학 비유로 정리하면: 학습률은 약물의 **용량 적정(dose titration)**, 옵티마이저는 **투여 경로(route)**, 스케줄러는 **용량 감소 프로토콜** 입니다. 적정 용량을 잘못 잡으면 효과가 없거나(과소) 독성을 유발(과대)하듯, 학습률 결정이 학습의 성패 대부분을 결정합니다.

### 과적합과 정규화 — 외삽 능력의 본질

**과적합(overfitting)** 은 모델이 훈련 데이터의 노이즈까지 외워버려 새 데이터에서 성능이 떨어지는 현상입니다. 약학적으로는 한정된 임상 코호트의 특이 반응에 모델을 끼워 맞춰 일반 환자에게 일반화되지 않는 상황과 같습니다.

방어책으로는 다음이 표준입니다.

- **L2 정규화(weight decay)**: 큰 가중치에 패널티
- **Dropout**: 학습 중 일부 뉴런을 무작위로 끔 → 앙상블 효과
- **데이터 증강(augmentation)**: SMILES 무작위 표현, 분자 회전 등
- **조기 종료(early stopping)**: 검증 손실 상승 시 학습 중단

## 작동 원리와 아키텍처

### 학습 파이프라인의 표준 흐름

```
[원시 데이터] → 정제·중복 제거·표준화
   ↓
[Scaffold Split] → train / val / test
   ↓
[전처리] → 정규화·증강·텐서 변환
   ↓
For epoch in 1..N:
    For batch in DataLoader(train):
        ŷ = model(x)              # forward
        loss = L(ŷ, y)            # 손실 계산
        loss.backward()           # 역전파
        optimizer.step()          # 가중치 갱신
    val_loss = evaluate(model, val)
    if val_loss > best + patience: break   # early stopping
   ↓
[테스트 평가 1회] → 보고용 지표
```

### 바이브코딩 시 AI에게 명시해야 할 항목

```
학습 사양 명세서:
1. 데이터셋     : 출처, 라벨 단위, 표준화 방법(예: ChEMBL pIC50, RDKit 표준화)
2. 분할 방식    : scaffold_split, random_seed=42, train/val/test = 8/1/1
3. 손실함수     : MSE (회귀) 또는 weighted BCE (불균형 비율 명시)
4. 옵티마이저   : AdamW(lr=3e-4, weight_decay=1e-2)
5. 스케줄러     : warmup 5% + cosine decay
6. 정규화       : dropout=0.2, early_stopping(patience=10)
7. 평가지표     : R²·RMSE (회귀) / AUC-ROC·MCC (분류) + scaffold split 기반
8. 재현성       : seed 고정, 데이터 hash 기록, 모델 체크포인트 저장
```

이 8개 항목을 빠뜨리지 않는 것만으로도 바이브코딩의 결과 품질이 크게 달라집니다.

## 신약개발 적용

- **Therapeutics Data Commons (TDC, Harvard)**: 60+ 신약개발 과제에 대해 scaffold split을 표준 분할로 채택했습니다. 동일 모델이 random split에서 AUC 0.92, scaffold split에서 0.71로 보고되는 사례가 흔하며, TDC는 후자를 공식 점수로 채택합니다.
- **Insilico Medicine Pharma.AI**: 다목적 분자 생성 시 강화학습 보상에 활성·합성 가능성·ADMET 점수를 가중 합산합니다. 보상 가중치(손실 함수의 일종)를 잘못 설정하면 모델이 합성 불가능한 분자에 수렴하므로, 약학적 우선순위에 맞춘 가중치 튜닝이 핵심 역량입니다.
- **AstraZeneca의 hERG 예측 모델 (2023, *J. Chem. Inf. Model.*)**: hERG 차단 양성 클래스 비율이 약 10%인 불균형 데이터에서 단순 BCE 대신 focal loss와 가중 샘플링을 결합해 MCC를 약 0.15p 개선했다고 보고했습니다. 손실함수 선택이 임상 안전성 예측의 신뢰도를 직접 좌우한 사례입니다.
- **MolFormer-XL (IBM, 2022)**: AdamW + warmup + cosine 스케줄을 11억 SMILES 사전학습에 사용해 안정적 수렴을 달성했습니다. 옵티마이저·스케줄러 선택이 대규모 사전학습의 성패를 결정합니다.

## 창업 관점

Phase 1 단계에서는 간략히만 짚습니다. 약학 전공자가 만드는 모델·서비스의 차별화 포인트는 화려한 아키텍처보다 **올바른 분할·손실함수·평가지표를 선택하는 약학적 안목** 에서 나옵니다. 동일한 GNN이라도 scaffold split + 임상적으로 의미있는 손실 가중을 적용한 모델은 실험실에서 재현되고, 그렇지 않은 모델은 데모 데이터셋에서만 빛납니다. 구체적 비즈니스 모델은 Phase 5에서 다룹니다.

## 오늘의 과제

1. **분할 방식 비교 리서치 (40분)**: TDC(Therapeutics Data Commons) 웹사이트(tdcommons.ai)에서 ADMET 과제 중 하나(예: Caco-2, hERG)를 골라, 동일 과제에 대해 random split과 scaffold split의 SOTA 점수를 비교하고 차이의 원인을 1쪽으로 정리합니다.
2. **손실함수 의사결정 표 작성 (40분)**: 신약개발 과제 5개 — ① pIC50 회귀, ② hERG 차단 분류(양성 10%), ③ 다중 ADMET 동시 예측, ④ SMILES 생성, ⑤ 희귀 부작용 예측 — 각각에 대해 추천 손실함수와 그 이유, 그리고 잘못된 손실 선택 시 실패 시나리오를 표로 정리합니다.
3. **학습 사양 명세서 초안 (30분)**: 본인이 만들고 싶은 가상의 모델(예: "BBB 투과 예측 모델") 하나를 정해, 위 "학습 사양 명세서" 8개 항목을 모두 채운 1쪽 문서를 작성합니다. Phase 4의 바이브코딩 단계에서 이 문서를 그대로 AI에게 입력해 모델 구축을 시작할 수 있는 수준이어야 합니다.
