# Day 5: Week 1 종합 — AI 기술 지도와 신약개발 연결점

> 이전 학습에서 데이터·손실함수·최적화로 이어지는 학습 루프의 내부를 다뤘습니다. 오늘은 Week 1에서 다룬 AI·ML·DL·학습원리 네 축을 하나의 지도로 통합하고, 신약개발 파이프라인의 어느 단계에 어떤 기술이 어떻게 대응하는지 정리합니다.

## 개요

이번 주 5일은 AI 신약개발의 **언어**를 익히는 과정이었습니다. 첫째 날 AI·ML·DL의 위계를 정의했고, 둘째 날 지도·비지도·강화학습의 패러다임을 구분했으며, 셋째 날 단일 뉴런부터 Transformer까지 신경망 진화를 다뤘고, 넷째 날 데이터·손실·최적화로 학습 루프의 내부를 들여다봤습니다. 이 지식들을 흩어진 채로 두면 단순 용어집에 머물지만, 신약개발 파이프라인 위에 매핑하면 **어떤 기술이 어떤 임상 의사결정을 어떻게 지원하는가**라는 질문에 체계적으로 답할 수 있습니다. 오늘의 목표는 Week 1 학습 내용을 ① AI 기술 지도, ② 신약개발 파이프라인 매핑, ③ 약학 전공자의 차별 포인트로 통합 정리하고, Week 2(신약개발 파이프라인과 AI)로 넘어갈 다리를 놓는 것입니다.

## 핵심 개념

### Week 1 핵심 정리 — 4일을 한 장으로

| Day | 핵심 개념 | 신약개발 함의 |
|-----|---------|-------------|
| 1 | AI ⊃ ML ⊃ DL의 위계, 규칙 기반 vs 데이터 기반 | "logP > 5이면 흡수 불량" 같은 인공 규칙을 넘어 데이터에서 비선형 패턴을 학습 |
| 2 | 지도/비지도/강화학습 패러다임 | 라벨 충분 → 지도(IC50 회귀), 라벨 부족 → 자기지도(분자 사전학습), 다목표 최적화 → 강화학습(분자 생성) |
| 3 | MLP·CNN·RNN·GNN·Transformer의 진화 | 모달리티별 적합 구조: 그래프 → GNN, 서열 → Transformer, 3D 좌표 → SE(3) 등변(equivariant) 모델 |
| 4 | 데이터·손실함수·최적화 | scaffold split, 가중 손실, AdamW, 불확실성·AD 보고가 임상 신뢰성의 기본 |

이 네 축은 독립이 아니라 누적입니다. **패러다임(2일)** 을 정한 뒤 **아키텍처(3일)** 를 고르고, **학습 루프(4일)** 를 신약 데이터에 맞게 조율하는 것이 한 줄짜리 워크플로우입니다.

### 통합 의사결정 트리 — 신약개발 문제 → AI 설계 선택

신약개발 문제를 받아들고 AI 설계를 결정하는 표준 사고 흐름은 다음과 같습니다.

1. **무엇을 예측·생성하는가?**
   - 단일 물성(pIC50, logP, hERG) → 회귀/분류 지도학습
   - 분자 자체(de novo 생성) → 생성 모델 + 강화학습
   - 단백질-리간드 결합 → 구조 기반 지도학습 또는 동역학 결합
2. **데이터는 얼마나 있는가?**
   - 라벨 ≥ 10만 → 딥러닝 직접 학습 가능
   - 라벨 1천~1만 → 사전학습 모델(MolFormer, Uni-Mol) + 미세조정
   - 라벨 < 1천 → 전통 ML(Random Forest, XGBoost) 또는 few-shot
3. **모달리티는 무엇인가?**
   - 분자: SMILES(Transformer) / 그래프(GNN) / 3D(SE(3) GNN)
   - 단백질: 서열(ESM) / 구조(GVP, AlphaFold 임베딩)
   - 텍스트: LLM 기반(논문 요약, 가설 생성)
4. **임상적 의사결정에 어떻게 쓰일 것인가?**
   - go/no-go 기준 → 분류 + 비용민감 손실 + 신뢰구간
   - 우선순위 랭킹 → 순위 학습 + AD 필터링
   - 후속 실험 설계 → 능동학습(active learning)

이 트리에서 Week 1의 모든 개념이 한 자리씩 차지합니다. 약학 전공자의 가치는 **4번 단계(임상적 활용 맥락)** 에서 가장 강하게 드러납니다.

### 신약개발 파이프라인과 AI의 매핑 — 한눈에 보기

신약개발은 보통 5단계로 구분합니다: **타겟 발굴 → 히트 발견 → 리드 최적화 → 전임상 → 임상**. Week 1에서 다룬 기술을 단계별로 매핑하면 다음과 같습니다.

| 파이프라인 단계 | 핵심 AI 패러다임 | 대표 모델/기술 | Week 1과의 연결 |
|-------------|-------------|--------------|-------------|
| 타겟 발굴 | 비지도 + LLM | 그래프 임베딩, 문헌 마이닝 | Day 2 비지도, Day 3 Transformer |
| 히트 발견(가상 스크리닝) | 지도학습 + 랭킹 | QSAR, GNN, 도킹 점수 | Day 2 지도, Day 3 GNN, Day 4 손실 설계 |
| 리드 최적화 | 강화학습 + 다목표 | REINVENT, Pocket2Mol | Day 2 RL, Day 4 다과제 손실 |
| 전임상(ADMET) | 지도 + 멀티태스크 | DeepTox, ADMET-AI | Day 4 가중 손실, 불확실성 |
| 임상 설계 | 지도 + 인과추론 | TrialGPT, 환자 매칭 | Day 1 AI 정의, Day 3 LLM |

이 표는 Phase 1 Week 2에서 단계별로 한 번 더 깊이 다룰 예정이며, 오늘은 "지도 위에 점이 어디에 찍히는가" 수준에서 익히는 것이 목적입니다.

## 작동 원리와 아키텍처

### 통합 워크플로우 — Week 1 개념을 한 그림에

신약개발 AI 시스템은 거의 모든 경우 다음 7단계로 분해됩니다. Week 1에서 학습한 개념들이 각 단계에 자리잡습니다.

```
표준 AI 신약개발 워크플로우 (Week 1 개념 매핑):

1. 문제 정의      : 무엇을 예측/생성? (Day 1: AI의 정의, Day 2: 패러다임 선택)
2. 데이터 수집    : ChEMBL/PDB/PubChem (Day 4: 데이터의 질이 천장)
3. 표현 선택      : SMILES / 그래프 / 3D (Day 3: 모달리티별 아키텍처)
4. 모델 선택      : RF / GNN / Transformer (Day 3: 아키텍처 진화)
5. 학습 루프      : forward → loss → backward → update (Day 4: 학습 원리)
6. 평가·검증      : scaffold split + 5시드 + AD (Day 4: 일반화)
7. 의사결정 통합   : go/no-go, 우선순위, 신뢰구간 (Day 1·2: 약학 맥락)
```

### 약학 전공자가 차별적으로 개입할 수 있는 지점

각 단계에서 약학 지식이 **단순 비유가 아니라 실제 의사결정에** 들어가는 지점은 다음과 같습니다.

| 단계 | 비약학 데이터 사이언티스트의 한계 | 약학 전공자의 개입 |
|------|--------------------------|----------------|
| 1. 문제 정의 | "AUC가 높은 모델"이 목표 | 임상 단계별 go/no-go 기준에서 출력값을 거꾸로 정의 |
| 2. 데이터 수집 | 라벨이 있으면 그냥 사용 | 어세이 조건(완충액, 세포주, 농도 범위)의 임상적 일관성 검증 |
| 3. 표현 선택 | 성능 위주 | pharmacophore·이성질체·tautomer가 손실되지 않는 표현 우선 |
| 4. 모델 선택 | 최신 SOTA 추종 | 데이터 크기·외삽 영역에 맞춘 보수적 선택 |
| 5. 학습 루프 | 표준 MSE/BCE | hERG 위양성·위음성 비대칭, 활성 영역 가중 |
| 6. 평가·검증 | random split, ROC-AUC | scaffold/temporal split, 임상 의미 있는 임계값에서의 metric |
| 7. 의사결정 통합 | "예측값 0.85" 출력 | "AD 내부, 신뢰구간 ±0.2, hERG IC50 < 10μM 기준 양성" 통역 |

> 약학 전공자의 경쟁력은 모델 알고리즘에 있는 것이 아니라, **임상에서 무엇이 의미 있는가**라는 질문을 모든 단계에 주입하는 능력에 있습니다.

## 신약개발 적용

Week 1 개념이 어떻게 결합되어 실제 성과로 이어지는지, 대표 사례 세 가지로 확인합니다.

- **Insilico Medicine — INS018_055 (특발성 폐섬유증)**: Pharma.AI 플랫폼에서 PandaOmics(타겟 발굴)·Chemistry42(분자 생성)·InClinico(임상 예측)를 결합. 강화학습 기반 분자 생성(Day 2)과 다과제 손실(Day 4)로 18~30개월 만에 임상 후보를 발굴해 2023년 Phase II 진입을 보고했습니다. 한 모델이 아니라 **여러 패러다임의 조합**이 핵심입니다.
- **Recursion Pharmaceuticals — Phenomics 플랫폼**: 세포 이미지 수백만 장에 자기지도 학습(Day 2)을 적용해 표현형 임베딩을 구축, BioHive-2 슈퍼컴퓨터로 화합물–표현형 매칭을 학습. 2023년 Roche·Genentech와 약 70종 표적에 대한 약 1.2억 달러 규모 협업을 발표했고, 2024년 Exscientia 인수로 화학 생성 능력을 보강했습니다 (확인 필요).
- **Isomorphic Labs — AlphaFold 3 기반 신약개발**: DeepMind 분사로, 단백질-리간드-핵산 복합 구조를 예측하는 AlphaFold 3(Day 3 Transformer 계열의 변형)와 분자 생성을 결합. 2024년 Novartis·Eli Lilly와 합산 약 30억 달러 규모의 다년 협업을 체결한 것으로 보도되었습니다 (확인 필요). 사전학습된 단백질 표현(Day 2 자기지도)이 다운스트림 신약개발 과제의 출발점이 되는 패러다임 전환을 보여줍니다.

## 창업 관점

Phase 1 단계에서는 간략히 짚습니다. Week 1 학습의 사업적 가치는 두 가지입니다. 첫째, **기술 지도가 그려져야 어떤 시장에 들어갈지 보입니다** — 가령 라벨이 풍부한 ADMET 시장(지도학습)과 라벨이 희소한 희귀질환 타겟 시장(자기지도+RL)은 진입 전략이 다릅니다. 둘째, **약학 전공자의 차별 포인트가 어디에서 발현되는지 확인하면 포지셔닝이 명확해집니다** — 위 7단계 표에서 약학 개입이 가장 강한 1·5·6·7번이 SaaS의 차별화 지점이자 Phase 5에서 다룰 비즈니스 모델의 핵심 축입니다.
