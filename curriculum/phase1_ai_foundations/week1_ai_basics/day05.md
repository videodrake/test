# Day 5: Week 1 종합 — AI 기술 지도와 신약개발 연결점

> 이전 4일간 AI/ML/DL의 위계(Day 1), 학습 패러다임 세 축(Day 2), 신경망 아키텍처의 진화(Day 3), 학습 루프의 세 기둥인 데이터·손실·최적화(Day 4)를 다뤘습니다. 오늘은 이 네 조각을 **하나의 기술 지도(tech map)** 로 통합하고, 그 지도 위에 신약개발 파이프라인의 각 단계를 정확히 겹쳐 두어 다음 주(신약개발 파이프라인 학습) 진입을 준비합니다.

## 개요

Week 1은 "AI가 무엇이고 어떻게 학습하는가"를 다뤘습니다. 오늘의 종합은 단순한 요약이 아니라, **선택 기준** 을 만드는 작업입니다. 어떤 신약개발 문제에 어떤 패러다임·아키텍처·평가 프로토콜을 짝지을 것인가를 표 한 장으로 답할 수 있어야, Week 2 이후의 심화 학습이 단편 지식으로 흩어지지 않습니다. 학습자가 약학 전공 + 풀스택 창업자라는 정체성을 유지하려면, 기술 지도 위에 **임상 의사결정 맥락** 을 항상 포개어 두는 습관이 필요합니다. Phase 1의 마지막 두 주차(Week 2~4)는 파이프라인·분자 표현·데이터/평가를 차례로 다루므로, 오늘은 그 길목에서 Week 1의 개념들이 어떻게 재등장할지 미리 표지판을 세워 두는 시간입니다.

## 핵심 개념

### Week 1 네 개념의 한 줄 재정의

오늘 자신만의 언어로 즉답할 수 있어야 하는 정의 네 가지입니다.

| 개념 | 한 줄 정의 | 약학 대응 |
|------|---------|---------|
| **AI** | 인간의 학습·추론을 컴퓨터로 구현하는 기술 전반 | 약전(전통)→임상 데이터 분석(AI)으로의 사고 전환 |
| **머신러닝(ML)** | 데이터에서 함수를 학습하는 방법론 | SAR 학습의 자동·대규모화 |
| **딥러닝(DL)** | 다층 신경망으로 특징과 함수를 동시 학습 | 분자 표현(원자→작용기→약리단)의 자동 구성 |
| **학습 루프** | 데이터·손실·최적화의 반복으로 파라미터 갱신 | 가설→실험→검증→가설 수정의 반복 |

### AI 기술 지도 — 5축 분류

신약개발에 등장하는 AI 시스템은 다섯 축으로 분류하면 길을 잃지 않습니다.

| 축 | 선택지 | 결정 기준 |
|---|------|---------|
| **데이터 신호** | 라벨/무라벨/보상 | 라벨 비용·풍부도, 목적이 예측인지 생성인지 |
| **입력 표현** | 기술자 / SMILES 토큰 / 그래프 / 3D 좌표 / 이미지 | 구조 정보 보존 필요도, 데이터 양 |
| **아키텍처** | MLP / CNN / RNN·LSTM / GNN / Transformer / Diffusion | 입력의 귀납 편향, SOTA 도달성 |
| **학습 전략** | 지도 / 자기지도+미세조정 / 강화학습 | 라벨 양, 외삽 요구, 다목적 최적화 여부 |
| **평가 분할** | 무작위 / 스캐폴드 / 시간 기반 / 외부 코호트 | 임상에서 요구되는 일반화 거리 |

이 5축은 Week 2~4의 거의 모든 토픽에서 다시 등장합니다. 분자 그래프(Week 3)는 표현 축의 한 점, MoleculeNet/TDC(Week 4)는 평가 분할 축의 표준입니다.

### 신약개발 단계 ↔ AI 기술 매핑

파이프라인의 각 단계에는 우세한 AI 기술 조합이 있습니다. Week 2에서 깊이 다룰 내용을 미리 한 장으로 묶어 둡니다.

| 단계 | 핵심 질문 | 주된 데이터 신호 | 대표 아키텍처/전략 | Week 1 어느 개념의 연속인가 |
|------|---------|--------------|----------------|----------------------|
| 타겟 발굴·검증 | 어떤 단백질을 노릴 것인가 | 무라벨 오믹스 + 텍스트 | SSL 임베딩 + LLM 그래프 추론 | Day 2 비지도/SSL |
| 히트 발견 (가상 스크리닝) | 결합 가능한 분자는 무엇인가 | 라벨(활성) + 단백질 구조 | GNN/Transformer 분류, AF3 도킹 | Day 3 GNN/Transformer |
| 리드 최적화 | 어떻게 개선할 것인가 | 라벨 + 보상 | 강화학습 + 다중 파라미터 최적화 | Day 2 RL + Day 4 손실 설계 |
| ADMET 예측 | 약물성은 어떠한가 | 다양한 작은 라벨 셋 | 다과제(multi-task) 지도학습 | Day 4 다중 손실·정규화 |
| 전임상·임상 예측 | 사람에서 어떻게 작동할까 | PK/PD 시계열, 임상 데이터 | RNN/Transformer + PK 모델 통합 | Day 3 시퀀스 모델 |

> **핵심 인사이트**: 같은 파이프라인 단계라도, 학습 신호(라벨/무라벨/보상)와 평가 분할의 선택이 모델의 임상적 가치를 갈라놓습니다. 약학 전공자의 우위는 "어떤 모델이 가장 똑똑한가"가 아니라 "어떤 모델이 이 임상 의사결정에 신뢰할 만한가"를 정의하는 데서 발생합니다.

### 약학 전공자가 반복해 마주칠 6대 의사결정

Week 2 이후 매 Day의 콘텐츠 뒤에는 결국 다음 여섯 질문이 깔립니다.

1. 이 모델은 **임상 어느 결정** 에 쓰이는가? (go/no-go, 우선순위, 안전성)
2. 라벨 분포가 **임상 현실** 과 일치하는가? (편향, scaffold 군집, 시간 의존)
3. 양성/음성 비용이 **비대칭** 인가? (위양성·위음성의 임상적 손실 차이)
4. 출력에 **불확실성** 이 동반되는가? (점추정만으로 의사결정 금지)
5. 약학적 **제약조건** 이 보상/필터에 반영되었는가? (합성 가능성, hERG, BBB)
6. 모델 자체가 **외삽** 을 하고 있는가? (적용 영역, 스캐폴드 외 일반화)

이 6개 질문은 본 프로그램을 관통하는 체크리스트로 사용합니다.

## 작동 원리와 아키텍처

### Week 1을 관통하는 단일 학습 시스템

서로 다른 4일치 내용을 하나의 시스템 다이어그램으로 묶으면 다음과 같습니다.

```
[원시 데이터]
   │  (Day 1: 데이터에서 패턴 학습)
   ▼
[표현]──── 기술자 / SMILES / 그래프 / 3D
   │  (Day 3: 아키텍처가 입력 구조를 결정)
   ▼
[인코더]── MLP / GNN / Transformer / Diffusion
   │
   ▼
[학습 신호]
   ├── 라벨   → 지도학습 (Day 2)
   ├── 무라벨 → 자기지도 사전학습 (Day 2)
   └── 보상   → 강화학습 (Day 2)
   │
   ▼
[학습 루프] (Day 4)
   1. forward → 예측
   2. loss    → 손실 계산 (선택은 임상 의사결정 종속)
   3. backward → 기울기
   4. update  → 옵티마이저(AdamW) + 정규화 + early stop
   │
   ▼
[평가]   scaffold/temporal split + AD + 불확실성 (Day 4)
   │
   ▼
[배포]   API + UI + 모니터링 (Phase 4 예고)
```

이 다이어그램의 모든 박스에는 약학적 판단이 끼어들 수 있습니다. 표현 단계에서는 어떤 입체화학 정보를 보존할지, 학습 신호 단계에서는 어떤 라벨을 신뢰할지, 손실 단계에서는 어떤 비용 비대칭을 반영할지, 평가 단계에서는 어떤 외삽을 측정할지가 모두 임상 맥락에 종속됩니다.

### 설계 결정 체크리스트 — 약학 전공자용 한 페이지

Week 2 이후 어떤 모델 논문을 읽든 다음 표로 분해해 평가할 수 있어야 합니다.

| 결정 | 흔한 오류 | 약학 친화 권장 |
|------|---------|-------------|
| 표현 | 단일 기술자만 사용 | 다중 표현 비교 + 구조 정보 유지 |
| 패러다임 | 라벨 부족에도 지도학습 단독 | SSL 사전학습 + 라벨 미세조정 |
| 분할 | random split 단일 보고 | scaffold + temporal split 병행 |
| 손실 | MSE 또는 BCE 기본값 | 비용 비대칭 반영(가중·Focal) |
| 옵티마이저 | 임의 학습률 1e-3 | AdamW + warm-up + cosine |
| 정규화 | 단일 시드 | 5 시드 + dropout + early stop |
| 출력 | 점추정만 | AD + MC Dropout / 앙상블 |
| 보고 | 평균만 보고 | 분산, 신규 scaffold 성능 동반 |

이 표는 Phase 1 종료 시점에 "AI 신약개발 모델을 평가할 수 있는 최소 도구함" 이 됩니다.

## 신약개발 적용

Week 1의 개념들이 결합되어 실제 의사결정을 갈랐던 사례를 세 가지로 정리합니다.

- **Insilico Medicine — INS018_055 (특발성 폐섬유증)**: 타겟 발굴(SSL+텍스트 마이닝) → 분자 생성(강화학습) → 우선순위(다과제 지도학습 ADMET) → 합성·실험 검증의 전체 사이클을 30개월에 달성했다고 보고했습니다. 2024~2025년 Phase IIa 진행 중으로, Week 1에서 본 세 패러다임이 한 파이프라인에서 어떻게 연결되는지 보여주는 대표 사례입니다.
- **AlphaFold 3 (DeepMind · Isomorphic Labs, 2024 *Nature*)**: Pairformer(Transformer)와 확산 모델을 결합해 단백질-리간드 복합체를 단일 네트워크로 예측. Isomorphic Labs는 2024년 Novartis·Eli Lilly와 총 약 30억 달러 규모 파트너십을 체결해 구조 예측 기술이 그 자체로 사업이 됨을 입증했습니다. Week 2 히트 발견·Week 3 표현·Week 4 평가의 모든 축이 한꺼번에 등장하는 사례입니다.
- **Recursion · Exscientia 모델 — 페노믹스+멀티모달 (2024~2025)**: 세포 이미지 SSL 임베딩 → 화합물-표현형 군집화 → 지도학습 표적 예측 → 강화학습 최적화의 하이브리드 파이프라인. 두 회사 모두 2024~2025년 빅파마와의 다년 협약을 갱신·확장하며 플랫폼 가치를 시장에 증명하고 있습니다(세부 조건은 회사 IR 자료 확인 필요).

기존 약학·통계 기반 접근과 Week 1에서 본 풀스택 AI 접근의 차이는 다음과 같습니다.

| 항목 | 1990~2000년대 QSAR/약학 통계 | 2020년대 AI 풀스택 |
|------|--------------------------|-----------------|
| 표현 | 사람이 고른 기술자 | 모델이 학습한 임베딩 |
| 신호 | 라벨만 | 라벨 + 무라벨 + 보상 |
| 아키텍처 | 선형/규칙/얕은 망 | GNN/Transformer/Diffusion |
| 평가 | 무작위 분할 단일 | 스캐폴드·시간 분할 + 불확실성 |
| 결정권자 | 통계학자 | 약학자가 임상 맥락으로 통합 |

## 창업 관점

Phase 1 단계에서는 짧게만 짚습니다. Week 1 종합 시점에서 약학 전공 창업자가 잡아 두어야 할 메시지는 단순합니다. **"기술 자체"는 빠르게 상품화되어 누구나 쓸 수 있게 되지만, "이 임상 결정에 이 모델 출력을 어떻게 신뢰할 것인가" 라는 평가·해석 레이어는 약학 전공자가 가장 잘 짤 수 있는 자산입니다.** 구체적 시장 규모, 비즈니스 모델, 경쟁사 분석은 Phase 5에서 본격적으로 다룹니다. 오늘은 다만 "모델 신뢰성 레이어" 라는 키워드를 머릿속에 등록해 둡니다.

## 오늘의 과제

1. **Week 1 정리 노트 1쪽 작성 (40분)**: AI/ML/DL 정의, 세 학습 패러다임, 4단계 학습 루프, 분할·손실·최적화의 핵심 결정 각 1줄씩을 자기 표현으로 적어 한 페이지에 압축합니다. 그 뒤 "이번 주에 가장 놀랐던 한 가지" 와 "Phase 2에서 가장 깊이 파고 싶은 주제 한 가지" 를 각 2-3줄 코멘트로 덧붙입니다.
2. **AI×신약개발 매핑 자가 점검 (40분)**: 신약개발 4단계(타겟 발굴, 히트 발견, 리드 최적화, ADMET 예측)에 대해 ① 데이터 신호, ② 적절 아키텍처, ③ 평가 분할 권장, ④ 약학 전공자가 가져갈 의사결정 포인트를 각각 1-2줄로 정리합니다. Week 2 Day 1을 읽기 전 반드시 완료합니다.
3. **선택 회사 1곳 심층 정찰 (50분)**: Insilico Medicine, Recursion, Isomorphic Labs, Exscientia, Schrödinger, Chai Discovery 중 1곳을 선택해 2024~2025년 공식 자료(블로그, 보도자료, IR) 2건을 읽고, ① 어떤 학습 패러다임 조합을 사용하는지, ② 차별화 주장(claim)은 무엇인지, ③ 약학 전공자가 보강할 수 있는 약점은 무엇인지를 1쪽으로 작성합니다. 이 자료는 Phase 5 경쟁사 분석에서 재사용합니다.
