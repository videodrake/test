# Day 5: Week 1 종합 — AI 기술 지도와 신약개발 연결점

> 이전 학습에서 데이터·손실함수·최적화로 이어지는 학습 원리와 scaffold split의 약학적 의미를 다뤘습니다. 오늘은 Week 1에서 다룬 AI/ML/DL 개념을 하나의 **기술 지도(technology map)** 로 통합하고, 각 요소가 신약개발 파이프라인의 어떤 단계와 연결되는지를 정리합니다. 다음 Week에서는 이 지도를 신약개발 파이프라인 위에 직접 겹쳐 보는 학습으로 넘어갑니다.

## 개요

Week 1은 **AI의 위계**(Day 1) → **머신러닝 패러다임**(Day 2) → **딥러닝 아키텍처**(Day 3) → **학습 원리**(Day 4)로 좁혀 들어온 깔때기 구조였습니다. 오늘 Day 5는 이 깔때기를 다시 펼쳐, 약학 전공자가 AI 신약개발 회사를 이해하거나 직접 설계할 때 사용할 **결정 트리(decision tree)** 를 만드는 시간입니다. 핵심 질문은 세 가지입니다. 첫째, 주어진 신약개발 문제에 어떤 패러다임·아키텍처·학습 전략이 자연스러운가. 둘째, 약학 지식이 그 결정에서 어떤 차별적 가치를 만드는가. 셋째, 이 기술 지도를 들고 Week 2의 신약개발 파이프라인을 어떻게 항해할 것인가. 이 결산을 통해 Phase 1의 첫 좌표를 명확히 찍습니다.

## 핵심 개념

### Week 1 개념 지도 — 4일을 한 장으로

```
[Day 1] AI ⊃ ML ⊃ DL
   │
   └──▶ [Day 2] ML 패러다임: 지도 / 비지도(SSL) / 강화
              │
              └──▶ [Day 3] DL 아키텍처: MLP · CNN · RNN · GNN · Transformer · Diffusion
                          │
                          └──▶ [Day 4] 학습 원리: 데이터 · 손실 · 최적화 · 정규화
```

이 위계의 핵심은 "위 계층의 선택이 아래 계층의 가용 옵션을 좁힌다"는 점입니다. 강화학습 기반 분자 생성이라는 패러다임을 선택하면 보상 함수 설계가 손실 설계의 자리에 들어오고, scaffold split 같은 평가 프로토콜은 패러다임이 무엇이든 공통적으로 작동합니다. 약학 전공자는 어느 계층의 의사결정이 임상적 판단을 요구하는지를 식별하는 능력이 핵심 자산입니다.

### 패러다임 × 아키텍처 × 신약개발 문제 매핑

| 신약개발 과제 | 자연스러운 패러다임 | 권장 아키텍처 | 핵심 함정 |
|--------------|------------------|------------|----------|
| pIC50·logP 회귀 (QSAR) | 지도학습 | GNN 또는 사전학습 Transformer | 라벨 부족, scaffold 일반화 |
| 무라벨 분자 표현 학습 | 자기지도(SSL) | Transformer (MolFormer 류) | 평가지표 정의의 모호성 |
| 단백질 구조 예측 | 지도 + 진화 정보 | Transformer + Diffusion (AF3) | 동적 유연성, 알로스테릭 |
| 분자 생성·최적화 | 강화학습 또는 생성 모델 | VAE / Diffusion + RL | 합성 가능성, 보상 설계 |
| 세포 이미지 표현형 | 지도 + SSL | CNN / ViT | 배치 효과, 도메인 시프트 |
| 임상 텍스트 마이닝 | 지도 + LLM 활용 | Transformer LLM | 환각, 출처 추적 |

표의 마지막 열 "핵심 함정"은 약학적 판단이 필수적인 지점입니다. scaffold 일반화, 합성 가능성, 동적 유연성은 모두 wet lab과 임상에서 모델 출력이 무력화되는 전형적 원인입니다. 모델러만 있는 팀은 RMSE를 낮추지만, 약학 전공자가 합류한 팀은 "낮춘 RMSE가 임상 의사결정에 유효한가"를 묻습니다.

### 약학 지식이 기술 지도에 개입하는 7개 접점

Week 1에서 누적된 약학적 개입 지점을 정리하면 다음과 같습니다. 이 목록은 향후 Phase별 학습에서 반복적으로 깊어질 골격입니다.

1. **데이터 큐레이션**: 어떤 어세이 결과가 비교 가능하고, 어떤 라벨 단위(IC50 vs Kᵢ vs EC50)가 통합 가능한지 판단
2. **분할 전략**: scaffold split·temporal split·target split 중 임상 외삽에 부합하는 것을 선택
3. **손실·보상 설계**: 위양성/위음성 비대칭, 활성 구간 가중, 다중 물성 트레이드오프 정의
4. **AD(applicability domain) 평가**: 모델이 외삽하는 화학공간의 임상적 의미 해석
5. **불확실성 활용**: 의사결정 임계값을 PK/PD·치료지수와 연동
6. **출력 해석**: 예측 활성을 임상 단계 go/no-go 기준에 맵핑
7. **규제 정렬**: FDA AI/ML SaMD, ICH M7, GxP 요건과 모델 검증 프로토콜 연결

### Week 1에서 자주 혼동되는 개념의 재정렬

| 혼동 쌍 | 핵심 차이 |
|---------|---------|
| AI vs ML | AI는 상위 개념(규칙 기반 포함), ML은 데이터로 학습 |
| 비지도 vs 자기지도 | SSL은 비지도의 특수 형태로, 입력 일부를 라벨로 재활용 |
| 분류 vs 회귀 | 출력이 이산 클래스인지 연속값인지 — 손실 선택의 출발점 |
| MLP vs GNN | MLP는 벡터 입력, GNN은 그래프 위 이웃 정보 통합 |
| Random split vs Scaffold split | 전자는 성능 과대평가, 후자가 임상 외삽에 가까움 |
| 점추정 vs 불확실성 | 점추정은 위험 평가에 부적합, 불확실성과 AD가 의사결정에 필요 |

## 작동 원리와 아키텍처

### 신약개발 AI 시스템의 표준 구성 (Week 1 통합 뷰)

```
신약개발 AI 시스템 — 표준 구성:
1. 데이터 계층     : ChEMBL/PubChem/PDB/사내 어세이 (라벨·무라벨 통합)
2. 표현 계층       : SMILES → 그래프 → 임베딩 (GNN/Transformer 인코더)
3. 사전학습 계층   : 무라벨 분자/단백질로 SSL (MolFormer, ESM 류)
4. 과제 계층       : 회귀/분류/생성/랭킹 헤드 + 적합한 손실함수
5. 평가 계층       : scaffold/temporal split + 다중 시드 + 불확실성
6. 의사결정 계층   : AD 필터 + 임계값 + 임상/규제 해석
7. 운영 계층       : API 서빙, 모델 버전 관리, 재학습 트리거
```

이 7계층은 Phase 2~4에서 각 계층의 깊이가 채워지는 골격입니다. Week 1은 1~5계층의 원리를 다뤘고, 6계층은 약학 전공자의 본진, 7계층은 Phase 4의 바이브코딩 영역으로 이어집니다.

### 문제 정의에서 평가까지의 결정 흐름

| 단계 | 핵심 질문 | 약학 전공자의 차별 포인트 |
|------|----------|----------------------|
| 문제 정의 | 무엇을 예측·생성·랭킹할 것인가 | 임상 의사결정에 직결되는 출력 정의 |
| 패러다임 선택 | 라벨 가용성과 형태는 | 어세이 비용·재현성 기반 라벨 전략 |
| 아키텍처 선택 | 입력 모달리티와 귀납 편향 | 모달리티별 약리학적 함의 이해 |
| 학습 설계 | 손실·정규화·옵티마이저 | 임상 비용 비대칭 반영한 손실 가중 |
| 평가 설계 | 분할·지표·불확실성 | 임상 외삽에 부합하는 평가 프로토콜 |
| 배포·해석 | 의사결정 통합 방식 | go/no-go 기준과 규제 정렬 |

## 신약개발 적용

Week 1 개념의 통합이 실제 회사·논문에서 어떻게 발현되는지를 사례로 확인합니다.

- **Insilico Medicine — INS018_055**: 자기지도 사전학습 + GNN 기반 활성 예측 + 강화학습 분자 생성 + 다중 물성 보상 설계가 한 플랫폼에 결합된 대표 사례. 특발성 폐섬유증(IPF) 후보 INS018_055는 타겟 발굴에서 임상 진입까지 약 30개월에 도달했고, 2024년 Phase II 결과가 보고되었습니다 (세부 수치는 임상등록 및 회사 공시 확인 필요).
- **Isomorphic Labs — AlphaFold 3 상업화**: AF3의 Transformer + Diffusion 결합 아키텍처를 약물 설계로 전환. 2024년 Novartis·Eli Lilly와의 파트너십(총 30억 달러 규모)은 "딥러닝 아키텍처 진화 → 신약 파이프라인 자산"의 전형적 가치 변환 경로를 보여줍니다.
- **Therapeutics Data Commons (TDC, Stanford)**: 70개 이상의 표준 과제와 분할 권장사항을 제공해 Day 4의 평가 프로토콜 원칙을 인프라로 구현. ADMET·분자 생성·임상 결과 예측을 한 곳에서 비교 가능하게 만들어, 학술·산업 간 재현성 격차를 좁혔습니다.
- **Recursion · Schrödinger**: 세포 이미지 기반 표현형 스크리닝(CNN/ViT)과 물리 기반 + 딥러닝 하이브리드 자유에너지 계산을 각각 제품화. 동일 기술 지도에서 다른 좌표를 점유한 두 기업의 대조는 "패러다임·아키텍처 선택이 곧 회사 정체성"임을 보여줍니다.

기존 약학 R&D와 Week 1에서 다룬 AI 통합 R&D의 구조적 차이는 다음과 같습니다.

| 항목 | 전통 약학 R&D | AI 통합 R&D |
|------|------------|-----------|
| 가설 생성 | 문헌·전문가 직관 중심 | 데이터 기반 + 표현 학습 |
| 후보 탐색 폭 | 수만~수십만 | 수억~수십억 (가상 스크리닝) |
| 평가 주기 | 어세이 기반, 수개월 | 모델 예측, 수시간~수일 |
| 의사결정 | 정성적 판단 우세 | 불확실성·AD 기반 정량 결합 |
| 약학 전공자 역할 | 실험·해석 중심 | 데이터·평가·해석 전 단계 개입 |

## 창업 관점

Phase 1 마지막 Day인 만큼, 본격적 시장 분석은 Phase 5로 미루되 Week 1 결산 차원에서 한 가지 관점을 제시합니다. AI 신약개발 회사는 일반적으로 ① **플랫폼형**(SaaS·API로 모델 제공), ② **파이프라인형**(자체 후보물질 개발), ③ **하이브리드형**(플랫폼+공동개발)으로 분류됩니다. Week 1 기술 지도 위에서 보면 플랫폼형은 1~5계층의 보편성에 베팅하고, 파이프라인형은 6계층(약학적 의사결정)에서 가치 대부분을 회수합니다. 약학 전공 풀스택 창업자의 자연스러운 시작점은 **6계층의 깊이로 차별화한 좁은 영역의 플랫폼·하이브리드** — 예컨대 특정 적응증·모달리티·ADMET 영역에 특화한 도구 — 가 될 가능성이 큽니다. 구체적 BM·시장 규모는 Phase 5 Week 1~2에서 정량적으로 다룹니다.

## 오늘의 과제

1. **나만의 Week 1 개념 지도 작성 (40분)**: 4일치 콘텐츠를 참고해 "ML 패러다임 ↔ DL 아키텍처 ↔ 신약개발 과제"를 잇는 한 장의 다이어그램(혹은 표)을 직접 작성합니다. Week 1에서 다룬 회사·논문 사례를 각 노드 아래 1-2개씩 매핑합니다. A4 1쪽 또는 디지털 화이트보드 스크린샷 1장.
2. **약학 개입 지점 자기 평가 (40분)**: 본문 "약학 지식이 기술 지도에 개입하는 7개 접점"을 참조해, 본인의 약학 지식이 강한 영역(상)·중간(중)·약한 영역(하)을 7개 접점별로 자기 평가하고, "하"로 평가된 접점을 보강할 학습 계획을 Phase 2~5 일정에 매핑합니다. 반 페이지 메모.
3. **Week 2 사전 리서치 (40분)**: 다음 Week 주제는 "신약개발 파이프라인과 AI"입니다. FDA 또는 EMA 자료에서 신약개발 단계별 평균 소요 시간과 성공률을 1차 출처로 1개 찾고, 그 단계 중 본인이 가장 흥미를 느끼는 단계와 이유를 3-5문장으로 적습니다. (예: 발굴, 전임상, 임상 1·2·3상, 시판 후 감시 중 택일)
